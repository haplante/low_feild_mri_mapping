# Low-Field MRI · B₀ Field Mapping Dashboard

Interactive dashboard for visualising and analysing robot-acquired B₀ field maps
of low-field MRI magnets — 2D slices, 3D volumes, homogeneity statistics and
scan–rescan repeatability.

**Open `index.html` in a browser and drop `.xlsx` field maps onto the left panel.**
No install, no build step. (Plotly and SheetJS are loaded from CDNs, so an
internet connection is needed the first time.)

To use the **⚡ Load all sample data** button you need the page served over
HTTP rather than opened directly (browsers block local file fetches over
`file://`). Two launcher scripts do this for you — see
[Running the dashboard](#running-the-dashboard) below.

## Repository layout

```
├── index.html            ← the dashboard (open this)
├── run_dashboard.bat     ← Windows launcher (serves + opens the dashboard)
├── run_dashboard.sh      ← macOS/Linux launcher (same, via bash)
├── data/                 ← field-map Excel files (synthetic samples; real
│                            scanner files are git-ignored, see .gitignore)
│   └── manifest.json     ← file list used by the "Load all sample data" button
├── scripts/
│   ├── generate_fake_fieldmaps.py   ← regenerates the synthetic samples
│   └── kill_old_servers.ps1         ← used by run_dashboard.bat to clear stale servers
├── old/                  ← previous notebook, dashboard and original file names
└── netlify_correlation_dashboard/   ← unrelated earlier project (style reference)
```

## Running the dashboard

**Windows** — double-click `run_dashboard.bat`. It closes any dashboard
server left running from a previous session, starts `python -m http.server`
in its own window, and opens the dashboard in your default browser.

**macOS / Linux** — run `./run_dashboard.sh` (or `bash run_dashboard.sh`) from
a terminal in the repo folder. It frees port 8000 if something is already
bound to it, starts `python3 -m http.server`, and opens the dashboard.
Requires Python 3.

Both scripts serve on `http://127.0.0.1:8000/index.html`. Close the server
window/process (or `Ctrl+C`) when you're done.

## File naming convention

```
<SCANNER>_DSV<diameter mm>_RES<grid step mm>_S<scan number>.xlsx
e.g.  RATSCANNER_DSV60_RES8_S01.xlsx
```

The dashboard parses this to group files by scanner and to detect
**scan–rescan pairs** (same scanner, DSV and RES — different `S` number).
Files that don't match the convention still load; scanner name falls back to
the filename and DSV/resolution are estimated from the data.

### Expected sheet format

First sheet, a header row containing `x`, `y`, `z` (mm) and `B` (Tesla)
columns — the robot export format (`time stamp, x, y, z, Bx, B`). Trailing
summary rows (`maximum`, `minimum`, `average`, `ppm`, `… MHz`) are ignored
automatically.

## Dashboard features

- **Field map tab** — stat tiles (mean B₀, Larmor ¹H, peak-to-peak & RMS
  homogeneity in ppm, SD), 3D view (measured points / isosurface shells /
  orthogonal planes, optional DSV sphere, optional robot mapping trajectory
  with start/end markers), 2D slice browser (axial / coronal / sagittal,
  interpolated heatmap / contour / raw points — drag the slider **or scroll
  the mouse wheel over the map** to step through slices), field distribution
  histogram with Gaussian fit (μ, σ, skew, kurtosis).
- **Statistics tab** — full per-dataset table (mean, SD, min/max, P–P, median,
  IQR, ppm homogeneity, RMS ppm, skew, kurtosis, Larmor), CSV export,
  homogeneity-vs-DSV curve, per-dataset deviation box plots.
- **Scan–Rescan tab** — auto-pairs rescans, matches points by coordinate, and
  reports bias, SD of differences, RMSD, 95 % limits of agreement and Pearson r,
  with Bland–Altman, correlation, diverging difference-map and overlaid
  distribution plots.
- **Global options** — units (mT / µT / ppm / MHz), colour scales, percentile
  colour clipping, marker size, point overlay.

## Sample data

`data/` ships 10 synthetic magnets generated by
`scripts/generate_fake_fieldmaps.py` (bore DSVs 20–140 mm, 46–99 mT). Three
scan–rescan pairs are included (NEOSCAN, MOUSEMAG, HALBACH64), and NEOSCAN
also has a higher-resolution 5 mm map of the same magnet. Rescans share the
true field of their magnet with a small B₀ drift and probe noise, so
repeatability statistics behave realistically.

Real scanner data (ERNIE2, RATSCANNER) is kept out of this repo — see
`.gitignore` — but works exactly the same way if you drop it into `data/`
locally and add it back to `manifest.json`.

Regenerate the synthetic set with:

```bash
python scripts/generate_fake_fieldmaps.py   # needs numpy, pandas, openpyxl
```

If the dashboard is served over HTTP (e.g. via `run_dashboard.bat` /
`run_dashboard.sh`, or a Netlify deploy), a **⚡ Load all sample data**
button appears that loads everything listed in `data/manifest.json` in one
click. Over `file://` browsers block local fetches, so drop or browse the
files instead.
